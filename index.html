"while" loop
Now let's move on to a new topic called "while". This is a loop. Through it we can do one or more actions as many times as we want and need. Now, let's make an example, and let's discuss this topic through it.
   
        include < iostream >
        using namespace std;
        int main (){   
            
            int num, i=0; 
            cin>>num; 
            while(i < 10){ 
                num = num + 1; 
                i=i+1; 
            } 
            cout < < num; 
            return 0;
        
        }
The "while" loop is written like this:
        while("logical representation"){
            "action 1"
            "action 2"
            "action 3"
        }
and so on. (actions). We tell the computer that as long as i (i is zero) is less than 10, to add one to number. Then along with that, so that the cycle does not end and repeat this action endlessly, we also add i to one. That is, if we enter 0, it will give us 10 as an answer, if we enter 5, it will give us 15 as an answer.
Now let's write another program subject to the following condition: We enter a single natural number. We calculate how many divisors it has. Let me show you this program and then discuss.

    
        #include < iostream >
        using namespace std;
        int main (){

            int number, i=1, divisors=0;
            cin>>number;
            while( i <= number){
            if (number%i==0){
                divisors=divisors+1;
            }
                i=i+1;
            }
            cout << divisors;
        
            return 0;
        }
What if you find out what's going on? Don't worry, I'll explain. That is, first, as usual, to enter the number. Then we assign 1 to the counter named i. You will understand why later. We also declare an integer called "divisors". Here we will store the number of dividers. We know how to enter, we entered the number. Then the cycle begins. as long as i is less than number. Then we have this if operator: if(number%i==0)divisors=divisors+1; If you're good at math, you'll immediately understand what's going on. That is, if the number is divisible by i without a remainder, that is, it is a divisor of the number, and therefore we should add one to "divisors".




“for” loop
We already know the "while" loop. The "for" loop is very similar to it, but sometimes more convenient. Let's see how it works.
Two completely identical codes are presented. One is written with the familiar "while" loop, and the other with the "for" loop, which we should get to know today.
        //first:
            #include < iostream > 
            using namespace std; 
            int main (){ 
                
                int num, i=0; 
                cin>>num; 
                while(i < 10){ 
                    num = num + 1; 
                    i = i + 1; 
                } 
                cout < < num; 

                return 0;
            
            }
        //second:
            #include < iostream > 
            using namespace std; 
            int main(){ 
                
                int num; 
                cin>>num; 
                for(int i=0; i < 10; i++){ 
                    num = num + 1; 
                } 
                cout < < num;  

                return 0;
            
            }
In the first code we see an example executed with a while loop. in the second with "for". There is no difference in these programs if you know. As I told you before, "for" is more convenient in some cases. This is one of the cases. I will explain how it is. Let's look at the "while" entry first:
        int counter;
        while(boolean instance){
            "actions";
            "increase the counter by one";
        }
Now let's look at the "for" entry:
for(int "Counter"; "Boolean Expression"; "Increase Counter"){
    "actions";
}
If you look, in "for" 3 are combined into one. (int counter; , a logical representation, and increment the counter by one or as many as you want.) (When we write "i++" in the second code above, it is the same as "i=i+1".)
The "for" loop is a loop like the "while" loop, it doesn't do anything different, and it won't change anything, it's just a shortened version of the "while" loop. If you understand what the first code of "while" does (written above), our second code also does the same (also written above, with "for"). The difference is only in recording and the time to record it in the economy, although in some places "while" is more convenient, depending on what the task is.
We don't need to distinguish those examples anymore, because if you understand the program written with "while" loop, you will understand the program written with "for" as well. It's just written differently.
